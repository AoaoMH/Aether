# 关键策略

## 1. 请求体记录

在系统设置中, 可以修改请求体记录详情等级

| 日志等级 | 记录内容 |
| --- | --- |
| **Base** | 基本请求信息 |
| **Headers** | Base + 请求头 |
| **Full** | Headers + 请求体 |

## 2. 调度模式

1. 提供商优先
2. Key优先
3. 缓存亲和
4. 负载均衡
5. 固定顺序
6. 故障转移

## 3. 访问限制
*全局或 Key 级别的访问频率及配额限制*

## 4. 请求体压缩清理
*去除多余空格与不可见字符以节省 Token 操作*

## 5. 定时任务
*自动刷新模型列表及定时检查配额状态*
